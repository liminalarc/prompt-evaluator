## Problem
The dashboard has become low-value: day to day, the useful surfaces are the prompt list and (for
owners/admins) managing an org's members. Carrying a dashboard as the landing/primary nav adds a hop
and a surface nobody relies on. Shipped = a leaner IA — a **Prompts** nav link as the primary
destination, and org/member management reached via the **gear** on the organization — with the
dashboard removed (or demoted) rather than maintained.

## Value
As a user, I want to land where the work is (my prompts) and reach org/member management from the org
gear, so the app has one obvious path instead of a dashboard I skip past.

## Scope (exploratory — to be firmed at implement)
**In (to explore):**
- Make **Prompts** the primary nav destination / landing after sign-in.
- Reach org member management from a **gear** affordance on the organization (owner/admin — ties to 4.5).
- **Remove or demote the dashboard**: decide delete vs. keep-behind-a-link; migrate anything still
  worth surfacing (e.g. regression/trend signals) onto the prompt/version surfaces instead.
**Out:**
- Score-trend / regression analytics themselves (the data stays; this is about where it lives).
- Auth/permission changes (org gear reuses the existing owner-or-admin gate).

## Open questions
- What, if anything, on the dashboard is load-bearing and must be re-homed before removal?
- Does "Prompts landing" replace the current default route, or coexist as the default?
