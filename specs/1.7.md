## Problem
The registry (1.1) is a **flat list** of prompts. As prompts accumulate across our apps, owners
need to organize them. This spec adds a **folder tree** — top-level folders with **optional
subfolders** — so prompts are browsable by area. It is usable single-user now; it also
establishes the boundary that multi-user access is granted on: **permissions attach to the
top-level folder** (subfolders inherit), which spec 4.1 builds on. "Shipped" means a prompt can
live in a folder, be browsed by folder, and be moved between folders, through the SPA.

## Value
As a prompt owner I want to organize prompts into folders (with optional subfolders) so that I
can find and manage prompts by area, and so access can later be granted per top-level folder.

## Scope
**In:**
- A `Folder` concept forming a tree: name, optional parent. A prompt belongs to exactly one
  folder (with a default/root folder for otherwise-unfiled prompts).
- The **top-level folder is the permission unit** — the model must resolve a prompt's
  top-level ancestor cheaply (store the root ref or walk parents); 4.1 grants access there and
  subfolders inherit.
- API: create/rename/move folders; move a prompt to a folder; list prompts by folder; fetch the
  folder tree.
- Angular: a folder tree / sidebar in the prompts browse UI, create-folder, move-prompt, and a
  breadcrumb; the 1.1 list becomes folder-scoped.

**Out:** The permissions/auth themselves (**4.1** — this spec only fixes the boundary).
Automatic foldering from captured app data (1.2). Moving *versions* between prompts.

## Acceptance criteria
- [ ] Folders form a tree (top-level + optional subfolders); a prompt belongs to exactly one folder — proven by Domain + Infrastructure tests
- [ ] Folders and prompt-placement are creatable/movable/browsable via the API — proven by API tests
- [ ] The SPA browses prompts by folder (tree navigation + breadcrumb, move a prompt) — proven by unit tests + one e2e
- [ ] A prompt's top-level folder (the permission boundary) is resolvable/queryable — proven by a test (consumed by 4.1)
