## Problem
Eval-loop screens (dataset detail, prompt workspace) carry friction surfaced dogfooding `golf-dna`
([5.1](specs/5.1/5.1.md), findings U17–U20): add/reveal forms stay open after a successful submit (a 2.4
decision that reads as clutter and, for add-version, invites a duplicate); the manual add-test-case
`Origin` defaults to `Captured` and can't be corrected (immutable **and** no per-case delete → the only fix
is deleting the whole dataset); the `Prompt input` is a cramped 2-row textarea. Shipped = all fixed, tests
green, live on dev.

## Value
As a LitmusAI operator building datasets by hand, I want add-forms that collapse after submit, an honest
origin default I can recover from, and a roomy input field, so I assemble test cases fast without clutter,
mislabels, or cramped editing.

## Scope
**In:**
- **U17** — collapse all three reveal forms (add-version, add-test-case, add-scorer) to closed after a
  successful add; reopen on `+`.
- **U18** — the manual add-test-case `Origin` defaults to `Synthetic` (manual entry is hand-written).
- **U19** — a **per-test-case delete** on dataset detail, so a wrong origin is recoverable (delete + re-add).
- **U20** — `Prompt input` / `Upstream SLM output` / `Expected output` textareas taller (auto-grow, monospace).
- **U21** — eval-run per-test-case scorer badges render in a **stable order** on every row (sort by a stable
  key — scorer kind / creation order), so columns line up for scanning.
- **U22** — the Compare drawer's From/To pickers show the **actually-selected** version. A `[value]` on a
  native `<select>` whose `<option>`s come from `@for` is assigned before the options exist, so the box fell
  back to its first option (comparing v1→v2 but showing "v1" in the To box). Drive selection with `[selected]`
  per option instead, so the rendered choice tracks the signal.

**Out:**
- Editable `Origin` in place — rejected; `origin`/`input`/`seed` stay immutable (2.8/domain). U18 + delete
  solve the mislabel without breaking immutability.
- Markdown editor for inputs — rejected; inputs are data/JSON, not prose (2.10 gates the editor to
  Content/Rubric).
- All other eval-loop features (async runs [2.17](specs/2.17.md), structured judging [2.15](specs/2.15.md), …).

  option — unit test asserts the To select shows v2 when comparing v1→v2. (U22)

## Decisions
- 2026-07-22 — `Origin` stays immutable (delete + re-add, not edit) — consistent with `input`/`seed`
  immutability (2.8/domain); paired with the `Synthetic` default (U18) this recovers the mislabel cleanly.
- 2026-07-22 — Inputs stay plain textareas, not the 2.10 `MarkdownEditor` (inputs are data, not prose).
- 2026-07-22 — Reverses 2.4's "reveal forms stay open after submit" for these three forms; if rapid-multi-add
  is ever wanted, add an explicit "Add another" affordance rather than defaulting open.
- 2026-07-22 — U22 added mid-build at the user's request (dogfood: Compare drawer showed the wrong version in
  the To box). Root cause: `[value]` on a native `<select>` with `@for`-rendered options is set before the
  options mount; `[ngModel]` didn't reflect synchronously under the app's change detection, so the fix uses
  `[selected]` per option (re-evaluated each CD). Same pattern applies to any dynamically-optioned select.
