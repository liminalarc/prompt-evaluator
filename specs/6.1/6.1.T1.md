## Goal
Record an `AiUsageRecord` on **every** call through `EvalRunnerClient` (execute / judge / generate),
on success **and** failure, at the one seam all model calls funnel through.

- **Domain** — `AiUsageRecord` aggregate (no EF/HTTP leakage): model (as echoed), token counts
  (input, output, cache-create, cache-read), provider request id, UTC timestamp, feature/use-case
  (`subject-execution` | `llm-judge` | `synthetic-generation`), organization id, user id, status
  (`success` | `refusal` | `error`), latency ms, max-tokens requested. Cost fields land in T2 (nullable
  for now). Factory validates non-negative tokens/latency.
- **Application** — `IAiUsageRecorder` port (`RecordAsync(AiUsageRecord, ct)`); `AiUsageContext`
  (feature + org + user) value type threaded from `RunEvaluationHandler` (execute/judge) and
  `GenerateSyntheticFixturesHandler` (generate) as an explicit param, with an `AsyncLocal` ambient
  fallback so the judge path (via `LlmJudgeScorer`, which has no org/user) still attributes. Org id
  from `Prompt.OrganizationId`; user from `ICurrentUser`.
- **Infrastructure** — EF-backed `AiUsageRecorder` adapter; `DbSet<AiUsageRecord>` on `EvalDbContext`;
  `AiUsageRecordConfiguration : IEntityTypeConfiguration<AiUsageRecord>` in the
  `Infrastructure.Persistence.Configurations` namespace; EF migration adding the `ai_usage_records`
  table. Record inside `EvalRunnerClient` around each POST — capture status/latency/tokens on the happy
  path and in the `EnsureRunnerSuccessAsync` failure path.
- **eval-runner (Python)** — extend `judge` + `generate` responses to carry the full usage block
  (model echoed, input/output/cache-create/cache-read tokens, provider request id, status). Surface
  usage from `providers.structured()` (currently discards it) for both Anthropic + OpenAI adapters;
  read `cache_creation_input_tokens`/`cache_read_input_tokens` and the response `id`. `execute` already
  returns tokens/cost — extend it too with model echo + cache tokens + request id + status.
- **.NET DTOs** — widen `JudgeResponse`/`GenerateResponse` (and `ExecuteResponse`) plus the
  `JudgeVerdict`/`GeneratedFixtureData`/`PromptExecution` result records to carry the new usage fields.

## Done when
