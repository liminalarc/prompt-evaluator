## Goal
An Admin → AI Usage Angular feature mirroring the Model Catalog admin structure: dashboard + filter bar
+ calls table + export.

- **Feature** — `web/src/app/ai-usage/`: standalone component + one `AiUsageApiService` (relative
  `/api/admin/ai-usage` calls, never `HttpClient` in the component) + DTO mirror of the T3 responses.
- **Route** — `admin/ai-usage`, `canActivate: [authGuard, adminGuard]` in `app.routes.ts`; add an
  `isAdmin`-gated nav entry (org rail / app shell) alongside the other admin links.
- **Dashboard** — time-series (spend over time) + breakdown tables (by model / feature / user / org) +
  top-N.
- **Filter bar** — date range, model(s), feature(s), user(s), org(s), status — drives dashboard + calls
  table together.
- **Calls table** — paginated drill to individual calls with the T3 columns.
- **Export** — CSV download of the current filtered view.
- Reuse the shared kit (`PageHeader`, `Breadcrumb`, `Card`, `LoadingState`/`EmptyState`/`ErrorState`,
  `StatusBadge`, `.sb-table`, `sb-*` classes), `data-testid` hooks.
- Budget status surface (spend-vs-budget + threshold alerts) is added in T6, in this same area.

## Done when
- [ ] `admin/ai-usage` renders the dashboard (time-series + breakdowns + top-N), filter bar, and calls
      table against the API; guarded by `[authGuard, adminGuard]` — Karma unit tests (+ Playwright e2e

- [ ] Filters drive both dashboard and calls table; CSV export downloads the filtered view
