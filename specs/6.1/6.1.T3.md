## Goal
A read model over `ai_usage_records` that slices spend as granularly as possible, plus a drill-to-calls
list and CSV export. Read-only queries (own repository/query service; no domain mutation).

- **Filter** — any combination of: date range, model(s), feature(s), user(s), organization(s), status.
- **Aggregate / group by** — period (day / week / month), model, feature, user, org.
- **Per-slice metrics** — total cost, token totals (in / out / cache), call count, avg cost & tokens
  per call, success rate, latency p50 / p95.
- **Shapes returned** — a time-series (spend over time), breakdown tables (by model / feature / user /
  org), and top-N.
- **Calls list** — filterable, paginated table of individual calls (timestamp, feature, model, tokens,
  cost, user, org, status, latency, request id). Cursor or page/limit consistent with existing lists.
- **CSV export** — of the current filtered / aggregated view (streamed `text/csv`).
- **Endpoints** — new `Api/Admin/AiUsageEndpoints.cs`, minimal-API group `/api/admin/ai-usage`
  (`RequireAuthorization()`), DTOs at the edge. Authz gate is T4.

## Notes

  composition root next to `MapAdminUserEndpoints`/`MapModelEndpoints`.
