## Goal
Compute each ledger record's cost from a **versioned, config-backed** per-model pricing table and
**snapshot** it (cost + rate version) at write time, so a later price change never rewrites history.

- **Pricing table** — config-backed (appsettings section, e.g. `AiUsagePricing`) authoritative for the
  ledger: per model id, USD per million tokens for input / output / cache-write / cache-read, plus a
  table-level **rate version** string. Bound to a typed options class; a small `IUsagePricing` service
  resolves rates by model id.
- **Cost computation + snapshot** — on `RecordAsync`, compute cost from the record's tokens × the
  resolved rates and freeze `CostUsd` + `RateVersion` onto the `AiUsageRecord`. Unknown model → cost
  `null` + a `PricingMissing` flag (tokens still stored). Add `CostUsd`, `RateVersion`,
  `PricingMissing` to the domain aggregate + EF config + migration (extend T1's table).
- **Reconciliation (ledger-only, per plan decision)** — this table is authoritative for the ledger;
  eval-runner `_PRICING` and the catalog display price are untouched. Residual divergence → 6.2.

## Done when
- [ ] Cost is computed from the versioned config table and snapshotted (cost + rate version) on the
      record; changing config prices afterward does not alter historical records — unit test
- [ ] Unknown model → `CostUsd` null + flagged, tokens still stored — unit test
- [ ] Cache-write/cache-read rates factor into cost — unit test with cache tokens
- [ ] EF migration extends `ai_usage_records` with cost/rate-version/flag columns; backend (xUnit) green

## Notes
- Seed the config table from the known Anthropic + OpenAI rates (opus-4-8 5/25, sonnet-5 3/15,
  haiku-4-5 1/5, fable-5 10/50, gpt-4o 2.5/10, gpt-4o-mini 0.15/0.60). Cache rates per Anthropic:
  write ≈ 1.25× input (5-min), read ≈ 0.1× input — encode explicit per-model rates, not multipliers,
  so the table stays the single authority. Confirm current rates via the `claude-api` skill at build time.
- Depends on T1's `AiUsageRecord` + recorder. Recording call site stays in `EvalRunnerClient`;
  cost is applied in the recorder/domain, not the transport.
