## Problem
Dogfooding the eval loop end-to-end made it clear the screens read as flat and off-brand. An audit
found the design system is wired correctly (`brand-tokens/css` + `/components` imported, zero
hardcoded hex, a strong shared kit and an on-brand trend chart) ‚Äî the flatness comes from concrete
defects, not a missing system: (1) **Inter is never loaded** (app falls back to a system font),
(2) the brand **`.sb-table` is locally overridden** in `prompts.css` (de-branding every table),
(3) **almost no cards** ‚Äî pages are one flat 820px column of `title + bare table/form`, (4) most
toggles are **bare `.sb-btn`** (transparent, invisible), and (5) several referenced CSS classes
don't exist (`.filter`, `.run`, `.runs`, `.fixture-run`, `.result__*`) so those regions fall back to
raw browser defaults. On top of that, six point-findings from the walkthrough. This spec is a
cohesive overhaul (descends from 2.4): make the eval loop tight, fluent, card-based, and genuinely
on-brand. "Shipped" means the loop looks designed ‚Äî consistent cards, real typography, branded
tables, clear action hierarchy, legible run output ‚Äî in both light and dark.

## Value
As an operator I want the eval-loop screens to look polished and read clearly, so the tool is
pleasant to use as we build up the prompt testbed.

## Scope
**In:**
- **Foundations**
  - **Self-host Inter** (woff2 assets + `@font-face`) so `--sb-font-ui` resolves; works offline / in compose.
  - **Delete the `.sb-table` override** in `web/src/app/prompts/prompts.css` ‚Äî restore brand tables (uppercase eyebrow headers, 2px rule, `tabular-nums`) everywhere.
  - **Shared `Card`** in `src/app/shared/` (barrel-exported) standardizing on `.sb-card` (`__body`/`__foot`); replace analytics' bespoke `.card`.
  - **Button intent** ‚Äî convert bare `.sb-btn` toggles/actions to `--secondary`/`--ghost`/`--primary` so hierarchy reads.
  - **Dark-mode toggle** ‚Äî topbar control that flips `data-theme` on `<html>` and persists it (localStorage); `index.html` stops hard-pinning light. Every changed screen legible in light + dark.
- **Layout** ‚Äî widen data-dense pages beyond the single 820px column; multi-column card grids on the workspace, dataset, and analytics pages (the dashboard tile grid is the pattern); tighten vertical rhythm with the spacing scale.
- **Prompts list** ‚Äî folder **cards** (drop the `üìÅ` emoji for a token-colored badge/icon); card-ify the reveal forms; button intent.
- **Prompt workspace** ‚Äî each section (Version history, Compare, Datasets, Analytics) becomes a `.sb-card`; the **`+ Add version` control moves up beside Version history** and its reveal form lays out cleanly (findings 1‚Äì2).
- **Dataset detail** ‚Äî style the undefined regions: origin **filter** as `.sb-field`, the **run** trigger as a card/form, the **runs** list as a `.sb-table`; card-ify; add an optional **expected-output field** to the capture form (finding 3 ‚Äî the capture API already accepts `downstreamResult`, so web-only).
- **Eval-run detail** ‚Äî **pretty-print** the model output (JSON when it parses), show **latency / cost / tokens as labeled fields** and the **fixture input** alongside; style `.fixture-run`/`.result__*` (finding 4).
- **Behavior** ‚Äî the **Runs list refreshes** after a run completes (finding 5); switching orgs in the topbar **rescopes the Prompts page** live (finding 6).
- **Analytics** ‚Äî `.card` ‚Üí `.sb-card`; `app-empty-state` everywhere; consistency (the trend chart is already on-brand ‚Äî leave it, verify in dark).

**Out:** Backend changes (none needed). Deletion UI (spec 1.10). Regression-detector algorithm/messaging (spec 1.11). New product features beyond the above.
