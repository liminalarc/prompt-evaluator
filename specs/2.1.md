## Problem
Deterministic and LLM-judge scores need a human calibration layer. Without periodic human
labels we can't trust that the LLM judge tracks real quality, and some dimensions (nuanced
tone, domain correctness) need a person. We need a review queue where a human scores a
sampled set of run outputs, feeding results back as an `IScorer` result and calibrating the
judge.

## Value
As a reviewer I want to label a sample of eval outputs so that human judgement calibrates
the automated scorers and catches quality issues the judge misses.

## Scope
**In:**
- **Human scorer**: a third `IScorer` implementation whose results are supplied via the
  Angular review queue rather than computed automatically.
- Sampling: select a representative sample of a run's fixtures for review (configurable
  size/strategy).
- Angular **review UI**: show the fixture input (and upstream context), the prompt output,
  the rubric; capture the reviewer's score + notes; queue progress.
- Calibration surfacing: compare human scores vs. LLM-judge scores on the same fixtures
  (agreement / drift), feeding the analytics from 1.4.

**Out:** automatic judge re-tuning from calibration data (future). Advisory (2.2).

## Acceptance criteria
- [ ] A human score is recorded as an `IScorer` result and flows through the same analytics as other scorers — proven by Application/Infrastructure tests
- [ ] A run can be sampled for review with a configurable size/strategy — proven by unit tests
- [ ] The review UI presents input + upstream context + output + rubric and captures score + notes — proven by unit + e2e
- [ ] Human-vs-judge agreement is surfaced for reviewed fixtures — proven by unit tests + a dashboard view
- [ ] Queue progress is visible and resumable — proven by e2e
