## Problem
The fields where we author real markdown — a prompt version's **Content** (the system prompt; Golf/Stormboard
prompts are all `##`-headed markdown with lists) and an LlmJudge **Rubric** — are plain `<textarea>`s. There's
no structure while writing and no way to see the rendered result, so multi-section prompts are hard to scan,
review, and get right. **Shipped** = those fields use a reusable markdown editor with an optional **preview**
that renders the markdown (read-only), the raw text staying the source of truth.

## Value
As a prompt author I want a markdown editor with a preview toggle so I can write and review structured
prompt/rubric content comfortably instead of eyeballing raw text in a bare textarea.

## Scope
**In:**
- A reusable **markdown-editor** component: a textarea (source) + an **Edit ⇄ Preview** toggle that renders
  the markdown to sanitized HTML (read-only). Source text is authoritative; preview never mutates it.
- Wire it into the **version Content** field ("+ Add version" / new-version-seeded-from-latest) and the
  **LlmJudge Rubric** field (dataset scorers). Existing immutability rules unchanged (content stays
  immutable-by-add; the editor is an authoring aid, not a new mutation path).
- Rendering is **self-contained + sanitized** (XSS-safe): a vetted markdown renderer + HTML sanitizer.
**Out:** WYSIWYG / rich-text editing (source-markdown only); markdown in short description fields (dataset /
fixture / prompt descriptions — a later, optional extension); AI-assisted authoring → [[2.7]]; syntax
highlighting beyond markdown; a full split-pane live preview (a toggle is enough for v1 — revisit if wanted).

## Acceptance criteria
- [ ] Version **Content** and LlmJudge **Rubric** render through the markdown editor with an Edit⇄Preview toggle — unit + e2e
- [ ] Preview renders markdown to **sanitized** HTML; a `<script>`/`onerror` payload in the source does not execute — test
- [ ] Source round-trips unchanged (what you save === what you typed; preview is display-only) — test
- [ ] Immutability unchanged — content still can't be edited in place; the editor only appears on add/new-version — test
