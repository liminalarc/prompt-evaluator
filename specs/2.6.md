## Problem
2.5 needed the LiminalArc hero-gradient tokens (`--sb-hero-dark #062935`, `--sb-hero-mid #0d4a63`,
`--sb-hero-light #1a6b8a`) to build the navy brand topbar, but LitmusAI pins
`brand-tokens@v0.4.0`, which omits them (Stormboard's copy has them). 2.5 defined them **locally**
in `web/src/styles.css` as a documented stopgap. This spec closes the loop: get the hero tokens into
the shared `brand-tokens` package, bump LitmusAI to that release, and delete the local `:root`
override so there's a single source of truth again. "Shipped" means LitmusAI's hero chrome resolves
from the `brand-tokens` package, not a local stopgap.

## Value
As a maintainer I want the brand tokens to come solely from the shared package, so LitmusAI doesn't
carry a divergent local copy of brand values.

## Scope
**In:**
- In the **brand-tokens repo** (separate; coordinate there): add `--sb-hero-dark/-mid/-light` to
  `tokens.json`, rebuild `dist/`, cut a release/tag.
- In **LitmusAI**: bump the `brand-tokens` dependency (`web/package.json`, currently
  `github:liminalarc/brand-tokens#v0.4.0`) to the new tag; **delete the local hero-token `:root`
  block** in `web/src/styles.css`; verify the navy topbar/hero still renders identically.

**Out:** Other brand-tokens changes. Any visual redesign (that's 2.5).

## Acceptance criteria
- [ ] `brand-tokens` (the release LitmusAI pins) defines `--sb-hero-dark/-mid/-light` — proven by inspecting the resolved `node_modules/brand-tokens/dist/tokens.css`
- [ ] The local hero-token stopgap in `web/src/styles.css` is removed and the topbar still renders navy — proven by inspection + e2e
- [ ] `npm run build` + Playwright e2e green after the bump — proven by CI
