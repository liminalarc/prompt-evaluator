## Problem
One rubric over a **heterogeneous** dataset caps the fixtures it can't apply to, regardless of prompt quality.
5.1 R3: the round-debrief rubric demanded front/back-nine momentum + "2-3 patterns" — analyses a **sparse**
fixture (score + putts only) physically can't supply — so that fixture was capped ~0.62 no matter how good
the prose was, and the aggregate said "no gain" while the prompt genuinely removed a fabrication risk. The
walk worked around it by hand-authoring a *data-conditional* rubric; the tool should support that natively.

## Value
As an operator I want a rubric to score each fixture against **what its input can support** — so a mixed
rich+sparse dataset scores each fixture fairly and the aggregate reflects prompt quality, not a structural
mismatch between a criterion and a fixture.

## Scope
**In (pick a shape when built):**
- Let a rubric express **conditional criteria** ("if no nine-level data, don't expect/penalize momentum
  analysis; reward graceful sparse handling"), **or**
- **Per-fixture criteria / fixture metadata** that gates which criteria apply, **or**
- Tool guidance to **keep datasets homogeneous** (split sparse fixtures into their own dataset+rubric) — the
  cheap interim the walk used. (Related craft guidance: [[2.13]].)

**Out:** structured/severity judging (→ [[2.15]]); variance + rationale-diff (→ [[2.14]]); proactive dataset
*design* guidance (→ [[2.13]]); the advisory that suggests making a rubric conditional (→ [[2.2]]).
