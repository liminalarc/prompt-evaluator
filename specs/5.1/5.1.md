## Problem
LitmusAI is built and running on dev, but has never been pointed at real prompts from a
real app. We need to prove the whole loop end-to-end on production prompts from two sibling
apps — Cortex Golf and Stormboard — done **by hand** so the workflow is understood, not
automated away. Payoff is threefold: (a) learn the tool by using it, (b) measurably improve
those two apps' prompts, (c) leave a realistic, populated test-bed for future features.

## Value
As the LitmusAI maintainer I want a repeatable, hands-on workflow for onboarding an external
app's prompts and improving them, so that I learn the tool, ship better prompts back to
Cortex Golf and Stormboard, and have real data to build future features against.

## Scope
**In:** Discover + inventory every prompt in both apps (52 as of 2026-07-18: Golf 15,
Stormboard 37). Stand up each app as an org in LitmusAI (dev). Walk **every** prompt through
the per-prompt playbook below (register → dataset → baseline → improve → re-eval → backport →
log). All steps performed manually.
**Out:** Zatomic-backed registry — manual copy-in stays (→ 3.1). Automated prompt sync from
the source repos (none; copy by hand). Advisory automation (→ 2.2). New eval/review features —
use what exists; findings feed the backlog, not this spec.

## The per-prompt playbook (the repeatable steps)
Run these for each prompt, by hand:
1. **Study** — read the prompt in its home repo; note purpose, model, input/output shape,
   call site, and what upstream feeds it.
2. **Register** — copy the text verbatim into the app's org as Prompt + Version `v1` (baseline).
3. **Dataset** — assemble fixtures: capture real inputs (± outputs) from the app as the
   ground-truth corpus; fill coverage gaps with synthetic fixtures seeded from captures (1.2).
4. **Scorers** — pick scorer(s) fit for the output: deterministic / LLM-judge / human.
5. **Baseline** — run `v1 × dataset × scorers`; record the baseline score.
6. **Diagnose** — read the low scores / failures; form one hypothesis to improve.
7. **Improve** — author `v2`; run `v2 × same dataset × same scorers`.
8. **Compare** — check movement vs prior version (regression detection); iterate `v3+` until
   satisfied or diminishing returns.
9. **Backport** — copy the winning version back into the source app (or decline, with reason).
10. **Log** — one line of learnings per prompt: what moved the needle.
