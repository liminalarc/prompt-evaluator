## Problem
[[1.16]] tells a maintainer **which single version to ship** (the backport target — the highest-scoring
version above Current) but stops at the signal: the human still has to hand-carry that version's prompt
into the source app, eyeballing what changed. That's slow and error-prone (copy the wrong version, miss
a line). LitmusAI should **assist the backport**: for the target, generate a ready-to-use artifact so
the maintainer can apply it quickly and correctly. LitmusAI still **signals only** — it produces the
artifact; the human applies it in the source repo's own process (wired-in automation is future — 3.1).

## Value
As a maintainer, once LitmusAI flags the backport target, I want it to hand me exactly what to
change — the new prompt content and how it differs from what's live — as a copy-paste prompt or a
downloadable file, so I can backport in seconds without hand-diffing versions.

## Scope
**In:**
- A **"Prepare backport"** action on the target (from the 1.16 Deployment summary) that assembles a
  backport artifact for the prompt's **single backport target** version.
- **Two output options (choose one, for now):**
  - **Exact prompt** — the target version's full content, **copy-to-clipboard**, ready to paste into
    the source app's prompt. (Optionally prefixed with a short metadata header.)
  - **Markdown file** — a downloadable `.md` containing: prompt name, **from Current vN → target vM**,
    target model, the **full new content**, a **diff vs Current's content**, a per-scorer **score-delta
    summary** (target vs Current), and a short apply checklist.
- The artifact carries the context a human needs to apply it safely: prompt id/name, `Current vN →
  target vM`, target model, Current's commit SHA (if set), and the score deltas that justify the ship.
- **Diff is computed against Current's version content** (what's live), not the immediately-prior version.

**Out:**
- **Wired-in automation** — opening a PR / writing to the source repo or the Zatomic registry directly
  → [[3.1]] (this spec is the manual-artifact bridge until then).
- **Batch / multi-version** backport (a whole org's prompts at once) — future; 1.16 pins one target per
  prompt, and this assists that one.
- **Auto-detecting the source location** (which repo/file the prompt lives in) — the maintainer knows;
  capturing that mapping is a later concern (relates to `SourceApp` on the version + 3.1).

## Decisions
- 2026-07-19 — **Two output options now, wired-in later** (user): ship a **copy-paste prompt** *and* a
  **downloadable markdown** artifact for the manual backport; automated PR / registry write is deferred
  to [[3.1]]. Preserves 1.16's signal-only stance — LitmusAI never writes to a source repo.
- 2026-07-19 — Assists the **single** backport target (1.16's one-target-per-prompt rule), not every
  eligible version — one artifact, for the version you'd actually ship.
- 2026-07-19 — Diff + deltas are **vs Current** (what's live), since the point is "what do I change to
  go from what's shipped to the target."
- 2026-07-20 (plan sign-off) — **Backend assembles the whole artifact** (target content, the diff, the
  per-scorer deltas, the rendered markdown, a suggested filename); the web stays thin (clipboard +
  file download). Single source of truth, unit-testable, and hittable by hand (dogfooding). Reuses
  [[1.16]]'s `VersionStatusHandler` (resolve the target + Current) and 1.4's `ComparisonAnalyticsHandler`
  (per-scorer means/deltas), run once per dataset.
- 2026-07-20 (plan sign-off) — **Diff is a new pure `LineDiff` in Application** (LCS line diff →
  context/added/removed), mirroring the already-tested web `diff.ts`. The ~30-line TS↔C# duplication is
  accepted over a shared package — different runtimes, no abstraction warranted (no-premature-abstractions).
- 2026-07-20 (plan sign-off) — **Exact-prompt copies the raw target content, no metadata header** — what
  lands in the source app's prompt field is exactly the prompt (spec listed the header as optional; it's
  omitted for paste-safety; the metadata lives in the markdown artifact instead).
- 2026-07-20 (plan sign-off) — **Score-delta summary spans all the prompt's datasets**, one row per
  `(dataset × scorer)` shared between target and Current (delta = target − Current) — faithful to how
  [[1.16]]/[[2.9]] derive eligibility/the target across every dataset, not a single one.
