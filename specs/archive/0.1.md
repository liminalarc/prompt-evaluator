## Problem
Establish the minimal end-to-end skeleton so every subsequent spec builds on a working
system ‚Äî Angular, the .NET API, the Python eval-runner, and PostgreSQL all wired together
and reachable, even though they do nothing useful yet. One trivial prompt runs through the
whole path to prove the seams.

## Value
As a developer I want to run Prompt Evaluator locally and reach every layer end to end so
that subsequent specs add behavior to a proven skeleton rather than building in isolation.

## Scope
**In:**
- .NET solution with the four DDD projects (`Domain`, `Application`, `Infrastructure`,
  `Api`) plus per-layer test projects; DI wiring in `Api`.
- One trivial vertical slice: a hardcoded prompt string ‚Üí API endpoint ‚Üí eval-runner echo
  ‚Üí response, persisted as a single `EvalRun` row in Postgres.
- Python eval-runner FastAPI app with one `/health` and one echo/score endpoint.
- Angular app that calls the API and displays the round-trip result, styled with the
  shared **brand-tokens** package (`github:liminalarc/brand-tokens#v0.4.0`): import
  `brand-tokens/css` (+ optionally `/components`) globally, theme via `data-theme`, and
  reference `--sb-*` custom properties only ‚Äî no hardcoded hex.
- EF Core + Npgsql wired with one migration; Postgres reachable locally.
- **Full containerized runtime**: a `Dockerfile` per service (`api`, `eval-runner`, `web`)
  and a `docker-compose.yml` orchestrating all four services (`api`, `eval-runner`, `web`,
  `db`/Postgres) on a **shared Docker bridge network**, so services address each other by
  name (`api` ‚Üí `eval-runner:8000`, `api` ‚Üí `db:5432`) rather than `localhost`.
  - Healthchecks on every service; `depends_on` with `condition: service_healthy` so start
    order is correct (db ‚Üí eval-runner ‚Üí api ‚Üí web).
  - A named volume for Postgres data; migrations applied on API startup (or an init step).
  - `.env` / compose env wiring for the connection string, `EvalRunner__BaseUrl`, and
    `ANTHROPIC_API_KEY` (not needed for the echo slice, but the wiring exists).
  - `docker compose up` brings the whole stack healthy from a clean machine.
- Basic CI: build + lint + the (minimal) tests for all three layers; CI also builds the
  images and runs a compose smoke test (stack comes up healthy).

**Out:** real prompt registry (1.1), real datasets/fixtures (1.2), real scorers (1.3),
analytics (1.4). No LLM call yet ‚Äî the eval-runner endpoint just echoes. Production
orchestration (k8s/hosting) is out ‚Äî compose is the local + CI runtime only.

## Decisions
- 2026-07-11 ‚Äî Skeleton echoes rather than calling Claude, to prove the seams without
  coupling the first slice to model behavior or an API key.
- 2026-07-11 ‚Äî The skeleton stands up the **full containerized runtime** (docker-compose,
  all four services, shared network, healthchecks, named volume), not just per-process
  dev servers. Services address each other by name over the Docker network so the
  networking topology is proven from day one and every later spec inherits a working stack.
  Compose is the local + CI runtime; production orchestration is out of scope.

- 2026-07-11 ‚Äî `main` branch protection is live (PR-required, 0 approvals, linear history,
  no force-push/deletion) but with **no required status checks** yet. Slice 8 wires the CI
  jobs in as required checks; until then protection gates merges but nothing gates on green.

- 2026-07-11 ‚Äî Adopt the shared **brand-tokens** repo (`liminalarc/brand-tokens`, latest
  `v0.4.0`) as the web app's styling source of truth from the skeleton on, so Prompt
  Evaluator is visually consistent with the other apps and re-themes centrally. Consumed as
  a git dependency (no registry); components reference `--sb-*` custom properties only.
  Dependabot/Renovate can later auto-PR token bumps.

- 2026-07-12 ‚Äî Pinned to **Angular 20** (not the latest 22): the local/CI Node is 20.19
  and Angular 22 requires Node 22+. Revisit when the toolchain moves to Node 22.
- 2026-07-12 ‚Äî Pinned the whole **EF Core stack incl. `Microsoft.EntityFrameworkCore.Relational`
  to 10.0.9** in Api + Infrastructure. Npgsql 10.0.3 floats Relational to 10.0.4 while Core
  resolved to 10.0.9, so the published (container) API couldn't load Relational 10.0.9 at
  runtime. Explicit alignment ships the matching provider assemblies.
- 2026-07-12 ‚Äî The API applies EF migrations **on startup** (guarded on a configured
  connection string) rather than a separate init container, so a clean `compose up` is
  self-migrating and the bare `/health` test needs no database.
- 2026-07-12 ‚Äî The Angular app calls **relative `/api/...`**; the ng-serve proxy (dev) and
  nginx (compose) route it. No environment-specific base URL in the client.
