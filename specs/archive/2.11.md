## Problem
The reveal / expand-to-edit pattern (from [[2.4]] / [[2.8]]) opens inline forms across the app, but most offer
only a submit (Add / Save) — there's no consistent way to **abandon** an action you started. To back out you
either submit something you didn't want or navigate away, losing your place. **Shipped** = every inline
reveal/expand form has a **Cancel** that discards unsaved input and collapses back to the prior state, with
consistent placement and behavior everywhere.

## Value
As an operator I want a Cancel on every add/edit/reveal form so I can back out of an action without submitting
or losing my place.

## Scope
**In:** a **Cancel** control on every inline reveal/expand surface — collapses the form and **discards unsaved
field input**, returning to the summary row / closed toggle. Consistent label + placement (paired with the
submit) and, where feasible, **Esc-to-cancel**. Covers the known surfaces:
- **Prompts/workspace:** create-prompt (`+ New prompt`), add-version (`+ Add version`), edit version label,
  run-a-version (`+ Run a version`), create-dataset (`+ New dataset`).
- **Dataset:** add-fixture (`+ Add fixture`), generate-synthetic (`+ Generate synthetic`), edit-fixture
  metadata, add-scorer (`+ Add scorer`), edit-scorer (reconfigure).
- Apply via the shared reveal/expand affordance so new surfaces inherit Cancel by default.
**Out:** unsaved-changes *confirmation* dialogs (v1 discards silently — a later refinement if it bites);
undo of already-**committed** actions (that's deletion/lifecycle → [[1.10]]); cancelling an in-flight eval
**run** (server-side run cancellation is its own concern, not this UI-collapse).

## Decisions
- 2026-07-18 — v1 **discards silently** on Cancel (no "you have unsaved changes?" prompt) — add later only if
  users lose meaningful work.
- 2026-07-19 — **Shared affordance = a CSS+convention layer, not a component.** The reveal/expand forms are
  hand-rolled per surface (`show*` signal + a footer toggle button), not a single wrapper component, so
  "shared" here is the `.form-actions` pairing row (prompts.css, reused by all three screens via `styleUrl`)
  + a `cancelX()` handler convention + `(keydown.escape)` on each form. A wrapper component was rejected as a
  premature abstraction over heterogeneous forms (varied submit labels/disabled states). New surfaces inherit
  Cancel by copying the two-line pattern; documented in `web/CLAUDE.md`.
- 2026-07-19 — Cancel style is `sb-btn--ghost` (subtle back-out) placed **after** the primary submit, matching
  the existing submit-first layout.
