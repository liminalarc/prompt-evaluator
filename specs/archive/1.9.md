## Problem
The registry has a folder tree (1.7) but no top-level container to group folders or hang access
off, and the Prompts screen is hard to follow (always-open forms, a move dropdown on every row,
no sense of "where am I"). This spec introduces an **Organization** as the top of the hierarchy â€”
`Organization â€ş Folder tree â€ş Prompt â€ş {versions, datasets, analytics}` â€” and makes it the unit
multi-user access is later granted on (4.1). This **supersedes 1.7's "top-level folder is the
permission boundary" decision**: the boundary moves up to the organization, resolved directly from
`Prompt.OrganizationId` (no tree walk). The Prompts screen is reworked around the org context.

## Value
As a prompt owner I want my prompts grouped under an organization (with folders inside it) so that
everything has a clear home, I always know which org and folder I'm in, and access can later be
granted per organization.

## Scope
**In:**
- **`Organization` aggregate** (id, name) â€” a distinct top-level container that sits *above* the
  folder tree (an org is not itself a folder). A **seeded "Default" org**; existing folders and
  prompts migrate into it.
- **`Folder.OrganizationId`** (required) and **`Prompt.OrganizationId`** (required) + optional
  `FolderId`. The **permission boundary is `Prompt.OrganizationId`** (O(1)) â€” consumed by 4.1.
- **Invariant:** a prompt may only be filed into a folder in its **own** organization; a folder's
  parent must be in the same org (enforced in Application).
- **Org is required everywhere** â€” no default-org fallback on create; callers name the org. (The
  migration seeds Default + backfills; the UI/tests list orgs and pick one.)
- **API:** organization list / create / rename; org-scoped folder-tree and prompt listing; folder
  and prompt creation under an org; move-prompt validated to stay within its org.
- **Web â€” Prompts screen redesign:** an **organization dropdown** (all orgs one can access â€” all,
  for now). On selecting an org, the **main area is folder-navigable** â€” a breadcrumb
  (`Org â€ş â€¦ â€ş current folder`), the current folder's **subfolders** (click to descend), and the
  **prompts in the current folder** â€” a file-explorer feel, **not** a left sidebar tree. Create
  folder / create prompt are **collapsible** actions; move-prompt is a tidied per-row action.

**Out:**
- Auth/permissions themselves (**4.1** â€” this spec only relocates the boundary to the org and
  provides the "orgs you can access" list, unfiltered for now).
- Moving a prompt **between** organizations (a prompt stays in its creating org).
- Organization context on the `/datasets` and `/analytics` screens (unchanged this spec).
