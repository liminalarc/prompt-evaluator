## Problem
Three UX nits surfaced during the 5.1 v4-validation walk — each small, each real, none with a home:
- **R6** — the loud run-failure banner (R2, shipped [[2.12]]) is **page-level** at the top; triggering a run
  from a card lower on a long page puts the failure **off-screen** ("I didn't see anything happen"). Right
  message, wrong placement.
- **U15** — the workspace "Run a version" **recent-runs list labels runs by timestamp only**; it never
  resolves *which version / model* a run scored (the dataset page's Runs table already does — U14, [[2.8]]).
- **U16** — in **dark mode** the topbar **org-switcher dropdown options** render near-unreadable (dark-grey
  on dark); native `<option>` popup isn't dark-aware.

## Value
As an operator I want run failures I can't miss, run lists that say what each run was, and a readable dropdown
in dark mode — small frictions that add up across every prompt walk.

## Scope
**In:**
- **R6** — surface a run failure where the eye is: a **toast/snackbar** (persists until dismissed) and/or an
  **inline error by the run button**; optionally auto-scroll to the banner.
- **U15** — give the workspace recent-runs list the same **version + model** columns as the dataset Runs
  table (data already loaded via `p.versions`); consider a short run label.
- **U16** — make native dropdowns dark-aware (`color-scheme: dark` on the control/`:root` in dark theme, or
  explicitly token the select's colors). Check other native `<select>`s too.

**Out:** the reliability/scoring features (→ [[2.14]]/[[2.15]]/[[2.16]]/[[2.17]]); a broader design-system
pass (U16 may instead fold into [[2.6]] brand-tokens — decide at build time).

## Acceptance criteria
- [ ] A run failure is visible without scrolling (toast and/or inline-by-button) — unit + e2e [R6]
- [ ] The workspace recent-runs list shows version + model per run, not just a timestamp — unit [U15]
- [ ] The org-switcher dropdown options are legible in dark mode — review/e2e [U16]
