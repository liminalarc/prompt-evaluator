## Problem
The product was scaffolded as "Prompt Evaluator" — a generic name that collides with a
crowded field (Promptfoo, DeepEval, Vellum, Braintrust, …). We want a distinctive brand,
**LitmusAI**, applied everywhere before the codebase grows and a rename becomes costly.
"Shipped" means every brand-bearing string, path, service id, and the git repo itself read
`LitmusAI` / `litmus-ai`, with all four CI gates still green.

## Value
As the product owner I want the project renamed to LitmusAI before it grows, so the brand is
consistent everywhere and there's no costly rename later.

## Scope
**In:**
- Display brand `Prompt Evaluator` → `LitmusAI` (docs, web UI title/brand, eval-runner title).
- Service id `prompt-evaluator-api` → `litmus-ai-api` (`VersionEndpoints`, its test, CI smoke).
- Compose project name → `litmus-ai`.
- Internal infra ids: Postgres DB `prompteval` → `litmusai`; docker network `pe-net` → `litmus-net`
  (accepted: resets local Postgres dev volume).
- GitHub repo `liminalarc/prompt-evaluator` → `liminalarc/litmus-ai` + local `origin` URL.

**Out:**
- .NET namespaces — layer-named (`Domain`/`Application`/`Infrastructure`/`Api`) with no product
  prefix; nothing to rename.
- `pgdata` volume and internal package names `web` / `eval-runner` — generic, no brand leak.
- Historical GitHub Actions **run URL** in `specs/archive/0.1.md` — dated record; GitHub redirects.
- Local working-directory rename (`C:\Development\prompt-evaluator`) — can't be done from inside
  the running session; handed to the user as a manual step.

- 2026-07-12 — Name is **LitmusAI** (repo `litmus-ai`, service `litmus-ai-api`, compose

  avoiding the crowded `prompt-*`/`eval-*` naming space.

  `pgdata` left as-is (generic, unbranded).

## Deferred to the user (manual, can't be done from inside the running session)
- **Rename the local working directory** `C:\Development\prompt-evaluator` → `C:\Development\litmus-ai`
  (close the session, rename the folder, reopen). Git, remote, and history are unaffected.
- **Update your local dev DB**: the Postgres DB name is now `litmusai`. Update your user-secret
  connection string (`Database=litmusai`) or recreate the local DB; existing `prompteval` data is
  not migrated (accepted).
