## Problem
Organizations are the permission boundary ([[1.9]], [[4.1]]), but there's no admin surface to
manage the orgs **themselves**. Today an org is *created* as a side effect of the Prompts page
(creating one grants the creator Owner); there is no way to list all orgs, rename one, or delete
one from an admin view. [[4.3]] added an Admin folder (Users, Models) and manages a user's
*membership* in orgs, but deliberately left org-entity management to this spec.

Origin: split from [[4.3]] (user decision — 4.3 stays user-focused).

## Value
As a global admin I want to manage organizations — see them all, create, rename, delete, and manage
who belongs to each — from the Admin area, rather than only creating them implicitly on the Prompts
page.

## Scope
**In:**
- Admin **Organizations** page under the Admin folder ([[4.3]]): list all orgs with **member
  counts**; create, rename, delete. Delete cascades folders/prompts/datasets/runs ([[1.10]]) behind
  a **type-the-org-name-to-confirm** dialog that names what will be removed. Global-admin gated
  ([[1.13]]).
- Admin-gated `/api/admin/organizations` endpoints (list-all + counts, create, rename, delete),
  each gated by `IsGlobalAdminAsync` → 403 otherwise (the [[4.3]] `/api/admin/*` pattern). Reuses
  `Organization.Rename` and `IOrganizationRepository.DeleteAsync` (cascade, [[1.10]]).
- **Org-first admin member view** (user opted in): drill into an org to list its members (email,
  role), add/remove a member, set role — reusing `IUserDirectory` grant/revoke. Scoped to the
  global-admin surface (management, not org content).

**Permission semantics (explicit):** the global-admin flag gates the **management surface only**.
It grants access to every org's *management* (rename/delete/members) but **not** to that org's
*content* (prompts/datasets/runs) — those stay membership-gated on the data endpoints (unchanged).
An admin who wants an org's content adds **themselves** as a member (auditable), via this member
view. This spec does **not** touch data-endpoint gating.

**Out:**
- **Owner-facing** member management on the org's own detail page (owner-or-admin, member-scoped) —
  re-homed to [[4.5]] (played next).
- User admin-flag / membership-from-the-user-side / passwords — [[4.3]].
- Moving org *creation* off the Prompts page — the implicit create stays.
