## Problem
The [[2.19]] cohesion pass delivered its acceptance criteria (dates, chips, meaningful scores, D3
summarize-then-reveal, the D1 Drawer, the unified Compare, the D2 tabbed hub, the rename, variance
clarity, and the attention-centered dashboard). A handful of walkthrough findings were **explicitly
scoped out of 2.19** as "candidate own spec" or "future" pointers — this spec is their home so nothing
is silently dropped (per the no-silent-deferrals rule). Each is a user decision to schedule.

## Value
The remaining rough edges — a cramped source editor for long prompts/rubrics, a couple of minor
display gaps, and a nav-IA idea — are captured and homed, so they can be picked up deliberately rather
than lost in a closed spec.

## Scope (deferred items homed from 2.19)
- **W3 / W21 — Source editor for prompt content + LlmJudge rubric.** The add-version **Content** field
  and the scorer **Rubric** editor (both on the 2.10 markdown editor) are too small for a ~55-line
  prompt / ~15-line rubric, and want a **syntax-highlighting source editor** (e.g. CodeMirror), NOT a
  WYSIWYG — prompts need exact source fidelity (a WYSIWYG would mangle a literal `[DRILLS_JSON]`+JSON
  block). Enlarge + make resizable; ideally a real code editor. Upgrades [[2.10]]'s Edit side.
- **W28 — Eval-run model-output box is a small fixed-height scroll.** In the focused/expanded fixture
  view (the primary artifact) D3 permits showing more — give the model output more height / let it
  expand.
- **W31 — Score trend silently omits versions with no run.** (2.19 AD3.) A version with no run just
  vanishes from the trend line ("where's v4?"). Show a gap/marker for un-run versions. Needs a small
  backend change to surface no-run versions as null trend points + chart-gap handling; low value, so
  deferred out of 2.19's S2.
- **W39 — Org switcher as a left drawer/rail instead of a top dropdown (user idea, 2026-07-19).**
  Replace the topbar org `<select>` with a left slide-in drawer/rail listing **all accessible orgs**,
  switch one at a time (the `OrgContextStore` single-selection model is unchanged). Cleaner, more
  discoverable IA than a cramped dropdown; the user is validating the same pattern on another product.
  App-shell change (topbar + a shell-level nav drawer); the shared [[2.19]] `Drawer` gains a
  left-anchored `side` variant, or a persistent rail. Deserves its own validation — do not ride it into
  a large batch.

**Out:** anything already delivered in 2.19; findings that already have a spec home (see below).

- [x] W3/W21 — a roomy, monospace, resizable **source** editor for content + rubric (+ Regex/JsonSchema

- [x] W28 — roomier, resizable eval-run **model-output** view (the primary artifact).
- [x] W31 — versions with **no runs** are named beneath the score trend (the gap is explicit).

## Decisions
- 2026-07-19 — Created at the close of [[2.19]] to home its explicitly-deferred findings so the cohesion
  pass could be marked DONE without a silent deferral. Then the user scheduled the whole spec: **W39 first**
  (chosen from the menu), then W40 (surfaced mid-build), then W3/W21 · W28 · W31 through to ship.
- **AD — W3/W21 editor: enlarge + resize + monospace, not CodeMirror.** The finding's stretch goal was a
  syntax-highlighting code editor; delivered the roomy monospace *source* textarea (exact fidelity, tall,
  drag-resizable) instead of adding a CodeMirror dependency inside a ship batch. A full code editor
  (line numbers/highlighting) remains an option to revisit — not re-homed (the AC is met); flagged here.
- **AD — W39 org switch rescopes in place** (matches the old `<select>` contract), and **W40 org
  management is a per-org settings gear** on the active rail row (user's pick from the menu), with the org
  page following org switches so it's never stuck on a stale org.
- 2026-07-19 — **New follow-up surfaced (user):** the auto-assigned **"Default" org** is a placeholder that
  will never hold content; deleting it should be allowed, and a user with **no orgs** should see that
  reflected (a real empty state / onboarding) rather than a phantom default. → **spec'd separately as
  [[2.21]]** after this work (user: "let's spec that out when you are done").

## Scope (added during W39 build)
- **W40 — Organization detail as a tabbed page (Overview · Members).** The org rail's "Manage members"

  Overview · … · Manage access) by making `/organizations/:id` a tabbed page — an **Overview** (name,
