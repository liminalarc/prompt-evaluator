## Goal
Define budgets for a period and track spend against them with over-threshold alerts in the admin UI.
Tracking + alerting only — **no hard enforcement** (blocking a run over budget is explicitly out; a
follow-on).

- **Budget aggregate** — a **global** (workspace) budget + optional **scoped** budgets (per model /
  feature / org), for a period (monthly to start): limit amount (USD), period, optional scope
  key/value. Domain aggregate + EF config + migration.
- **Spend-vs-budget** — compute current-period spend for each budget's scope (reusing T3's aggregation)
  → amount used + % used.
- **Threshold alerts** — flag when spend crosses configurable thresholds (default 80% / 100%); surface
  in the admin UI as an alert/badge on the AI-usage dashboard.
- **API** — CRUD for budgets + a spend-vs-budget status endpoint, under `/api/admin/ai-usage`
  (global-admin gated per T4).
- **UI** — budget config surface + budget-status cards/alerts in the T5 area.
